# 大模型原理与技术课程 - 综合作业仓库（2025）

欢迎来到《大模型原理与技术》课程的作业提交仓库！

本仓库采用**单一综合作业**的形式，整个学期只完成**一个方向**的大项目。你们可以根据自己的研究兴趣或未来方向，从以下 8 个领域中**任选其一**，全程迭代开发一个完整、可复现、具有一定创新性的系统。

### 作业目标
- 深入掌握大模型的核心技术（预训练、微调、RAG、Agent、多模态、部署等）
- 完成一个从数据准备 → 模型构建 → 评估 → 部署的全流程项目
- 培养工程实践能力、代码管理能力和学术报告撰写能力
- 鼓励持续迭代，在学期内随时优化性能、修复问题、添加新功能

### 可选方向（请选择一个）
1. **自然语言处理（NLP）**  
   构建领域特定RAG问答系统  
   文件夹：`01-NLP/`

2. **图形图像及多模态**  
   构建多模态视觉-语言助手  
   文件夹：`02-Multimodal/`

3. **智能体及具身智能**  
   构建具身智能体（Embodied Agent）  
   文件夹：`03-Agent-Embodied/`

4. **无人机集群**  
   构建多无人机协同系统  
   文件夹：`04-UAV-Swarm/`

5. **智能软件工程**  
   构建AI驱动的软件开发助手  
   文件夹：`05-Software-Engineering/`

6. **数据处理及统计**  
   构建AI驱动的大规模数据分析代理  
   文件夹：`06-Data-Processing/`

7. **人工智能安全**  
   LLM红队攻击与防御研究  
   文件夹：`07-AI-Safety/`

8. **其他相关方向（自由选题）**  
   探索新兴AI方向（如AI for Science、可持续AI、多Agent协作等）  
   文件夹：`08-Other/`

**每个方向的详细题目描述、要求和评分标准，请仔细阅读对应文件夹下的 `README.md`。**

### 作业流程与时间安排
- **第1-3周**：确定选题，在对应方向的 GitHub Issues 中留言报名（格式：学号+姓名+选题），避免重复。
- **整个学期**：持续开发，随时提交代码更新（通过 Pull Request）。
  - 鼓励每周或每两周提交一次进度更新（可以是小改进、bug修复、新实验等）。
  - 每次PR都会被记录，作为过程性评价依据。
- **结课前一周（具体截止时间以课堂通知为准）**：提交最终版本报告 + 完整代码。
- **允许并鼓励迭代**：你们可以在整个学期内不断优化模型、数据、算法，提升性能，最后以最佳版本作为最终成绩依据。

### 提交方式（必须严格遵守）
1. **Fork 本仓库**到你自己的 GitHub 账号。
2. 在你选择的方向文件夹下，进入 `submissions/` 目录。
3. **代码提交**：
   - 强烈建议将完整代码放在一个**独立的公开 GitHub 仓库**（便于展示和复现）。
   - 在报告中必须提供该仓库的链接。
4. **报告提交**：
   - 将最终实验报告放入对应方向的 `submissions/` 目录。
   - 文件名严格格式：`学号-姓名-方向编号-方向简称.md` 或 `.pdf`  
     示例：
     - `20231234-张三-01-NLP.md`
     - `20231234-李四-03-Agent-Embodied.pdf`
     - `20231234-王五-08-Other-AIforClimate.pdf`（自由选题需加关键词）
5. **提交 Pull Request**：
   - PR 标题格式：`[最终作业] 学号 姓名 - 方向名称 - 项目标题简述`
     示例：`[最终作业] 20231234 张三 - NLP - 医疗领域RAG问答系统`
   - PR 描述请按照仓库根目录 `.github/pull_request_template.md`（如果有）填写，或至少包含：
     - 独立代码仓库链接
     - Demo 链接（如有）
     - 主要成果与创新点
     - 自评得分点

### 评分组成（满分100分）
- 过程性迭代（30分）：PR 提交频率、代码更新质量、问题解决能力
- 项目完整性与复现性（20分）：代码可运行、文档清晰、依赖说明完整
- 技术深度与创新（30分）：是否合理使用大模型技术、是否有自己的改进或创新
- 报告质量（15分）：结构清晰、图表完整、分析深入
- Demo 与展示效果（5分）：交互体验、视频/截图展示

### 注意事项
- 前7个方向必须严格按照对应 README 中的题目要求完成，第8个方向需在报告中详细说明选题理由与相关工作对比。
- 鼓励使用免费算力资源（Colab、Kaggle、AutoDL、天池等），并在报告中记录资源使用情况（可加分）。
- 所有代码和报告必须为本人独立完成（如组队需提前申请并说明贡献分工）。
- 涉及敏感内容（如越狱攻击）需遵守学术伦理，负责任地披露与讨论。
